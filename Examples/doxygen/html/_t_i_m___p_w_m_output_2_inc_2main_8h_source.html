68/* -----------------------------------------------------------------------
69TIM1 Configuration: generate 4 PWM signals with 4 different duty cycles.
70
71 In this example TIM1 input clock (TIM1CLK) is set to APB1 clock (PCLK1),
72 since APB1 prescaler is equal to 1.
73 TIM1CLK = PCLK1
74 PCLK1 = HCLK
75 => TIM1CLK = HCLK = SystemCoreClock
76
77 To get TIM1 counter clock at SystemCoreClock, the prescaler is set to 0
78
79 To get TIM1 output clock at 24 KHz, the period (ARR)) is computed as follows:
80 ARR = (TIM1 counter clock / TIM1 output clock) - 1
81 = 2666
82
83 TIM1 Channel1 duty cycle = (TIM1_CCR1/ TIM1_ARR + 1)* 100 = 50%
84 TIM1 Channel2 duty cycle = (TIM1_CCR2/ TIM1_ARR + 1)* 100 = 37.5%
85 TIM1 Channel3 duty cycle = (TIM1_CCR3/ TIM1_ARR + 1)* 100 = 25%
86 TIM1 Channel4 duty cycle = (TIM1_CCR4/ TIM1_ARR + 1)* 100 = 12.5%
87
88 Note:
89 SystemCoreClock variable holds HCLK frequency and is defined in system_stm32g0xx.c file.
90 Each time the core clock (HCLK) changes, user had to update SystemCoreClock
91 variable value. Otherwise, any configuration based on this variable will be incorrect.
92 This variable is updated in three ways:
93 1) by calling CMSIS function SystemCoreClockUpdate()
94 2) by calling HAL API function HAL_RCC_GetSysClockFreq()
95 3) each time HAL_RCC_ClockConfig() is called to configure the system clock frequency
96 ----------------------------------------------------------------------- */
97
